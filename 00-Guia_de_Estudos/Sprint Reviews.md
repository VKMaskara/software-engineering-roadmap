## 🧠 Etapa 2 — Revisão Ativa

### O que é um algoritmo?

-> Algoritmo é a quebra de um problema maior em pequenos passos, criando assim um passo a passo finito e com uma sequência lógica a fim de resolver o problema.

### Qual a diferença entre variável e constante?

-> Variável pode ser alterada durante a execução do código.

-> Constante mantém o seu valor durante toda a execução do código.

### Por que existe o operador %?

O operador `%` representa o resto da divisão. Podemos utilizá-lo, por exemplo, para identificar números pares e ímpares.

### Qual a diferença entre && e ||?

**&&** -> Operador lógico em que as duas condições precisam ser verdadeiras.

**||** -> Operador lógico em que apenas uma condição precisa ser verdadeira.

### O que é uma decisão composta?

-> Uma decisão composta é quando definimos no código um caminho para o caso da condição ser verdadeira e outro para o caso dela ser falsa.

# Sprint Review — Semana 02

## 🧠 Etapa 1 — Revisão das Anotações

### Teste de Mesa

**O que é um Teste de Mesa?**

-> Teste de Mesa é um método de leitura e execução manual de um código, basicamente a leitura e a interpretação de um algoritmo linha por linha.

**Qual é o objetivo de fazer um Teste de Mesa antes de executar um algoritmo?**

-> Entender o seu fluxo de execução e lógica para verificar e evitar a existência de erros.

**Qual foi a maior dificuldade que você encontrou ao fazer Testes de Mesa?**

=> Não tive grandes dificuldades. É basicamente pensar como se só coubesse uma única informação dentro de um espaço de memória (variável), e para uma entrar, a outra tem que ser sobrescrita.

---

### Fluxo de Execução

**O que é Fluxo de Execução?**

=> Fluxo de Execução é como se fosse um mapa de quais caminhos o nosso algoritmo deve percorrer com base nas condições. Se a condição for `true`, execute isso; se não, faça aquilo.

**Qual a diferença entre Fluxo de Execução e Teste de Mesa?**

=> A diferença está nas perguntas que eles respondem.

- **Teste de Mesa:**
  R: Que valor deveria estar nessa variável?

- **Fluxo de Execução:**
  R: Qual caminho foi percorrido até aqui?

**Por que o algoritmo continua executando após sair de um bloco `if/else`?**

- Pois o computador só para de executar quando o algoritmo acaba, e não quando o bloco `if/else` termina. Quando ele termina, é apenas aquele bloco que acabou.

---

### Depuração Manual

**O que significa depurar um algoritmo?**

-> É o ato de investigar e analisar o código a fim de encontrar e corrigir erros.

**Quais são os três principais tipos de erro que estudamos?**

-> Sintaxe, lógica e execução.

**Como o Teste de Mesa e o Fluxo de Execução ajudam na depuração?**

-> Eles nos ajudam a entender quais valores foram armazenados e com quais dados o computador está trabalhando, o que nos leva a compreender qual caminho está sendo percorrido.

## ⚡ Etapa 2 — Revisão Ativa

**Se uma variável muda de valor durante a execução, qual ferramenta ajuda a acompanhar essa mudança?**

- Teste de Mesa.

**Se um algoritmo segue o caminho errado, qual conceito ajuda a identificar isso?**

- Fluxo de Execução.

**Qual é a diferença entre um erro de sintaxe e um erro de lógica?**

Um erro de sintaxe é um erro de escrita no código. Um erro de lógica é quando o sistema não funciona como deveria por não seguir a regra de negócio.

**Por que um loop infinito é considerado um problema?**

- Porque isso pode travar navegadores, sistemas operacionais ou softwares.

**O que significa dizer que um algoritmo possui um fluxo sequencial?**

-> Ele é um algoritmo simples, sem a existência de condicionais, e segue uma linha reta de execução.

## 🚀 Etapa 3 — Desafio Integrador da Semana

Imagine um sistema de matrícula escolar com as seguintes regras:

- O aluno deve estar cadastrado.
- Deve ter a mensalidade em dia.
- Deve possuir vaga disponível na turma.
- Não pode estar bloqueado.

Um desenvolvedor escreveu o seguinte algoritmo:

```javascript
let cadastro = true
let mensalidade = true
let vaga = true
let bloqueado = true

if (cadastro && mensalidade && vaga){
    console.log("Matrícula liberada")
}else{
    console.log("Matrícula negada")
}

console.log("Registro da tentativa realizado")
```

Sem alterar o código inicialmente, responda:

**Qual é o erro presente no algoritmo?**

A variável `bloqueado` não está inserida na condicional, permitindo que alunos bloqueados tenham a matrícula liberada.

**Classifique o tipo de erro.**

Lógico, pois não segue a regra de negócio.

**Faça um Teste de Mesa.**

-> As variáveis seguem com o mesmo valor do começo ao fim.

**Descreva o Fluxo de Execução.**

```text

  |
  v
Declara as variáveis
  |
  v
Possui cadastro && mensalidade && vaga?
      |                                        |
     Sim                                      Não
      |                                        |
      v                                        v
 Matrícula liberada // Executado   Matrícula negada // ignorado
        \           /
         \         /
          v       v
   Exibir: Registro da tentativa realizado
            |
            v
           Fim
```

Eu leria o código linha por linha e o executaria mentalmente, utilizando os métodos de Teste de Mesa e Fluxo de Execução para me guiarem.
